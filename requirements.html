Requirements

Functional Requirements

Upload
  • As a user, I would like to upload MIDI files so that I can play them back.
  • As a user, I would like to upload files from my local file system so that I can work with the files I have saved on my device.
  • As a user, I would like to only see MIDI files in my file system while uploading so that it is easier to find the file I am looking for and I do not upload the incorrect one.
  • As a system/application, I would like to just allow MIDI files to be uploaded so that a file is not uploaded that I cannot play back.
  • As a system/application, I would like to limit uploaded files to be 50 KB and lower so that a file is not too long or large when it is uploaded.
  • As a user, I would like to be notified when a file is too large to upload so that I have adequate information about the issue and I can choose another file.
Playback
  • As a user, I would like to be able to play, pause, and stop playback so that I have control over the playback.
  • As a user, I would like to be able to skip to a measure in the song so that I do not have to wait for the playback to reach that point of the song.
  • As a user, I would like to be able to set the time signature and bpm for a song so that the speed of the song can be changed if needed.
  • As a user, I would like to be able to add labels to measures in a song so that I do not have to remember which measures contain the point of the song I want to jump to.
  • As a user, I would like to jump to the labels that I have created so that the labelled points are convenient to get to.
  • As a user, I would like to save any bpm or time signature changes locally so that when I open the file again my changes are still there.
  • As a user, I would like to save the labels I create locally so that the labels I create persist when I open the file again.
  • As a user, I would like to view where the playback is within the displayed measures so that I can follow which part of the track is playing.
  • As a user, I would like my label titles to be limited to 25 characters so that I can fit multiple on-screen.
  • As a user, I would like to be notified when I try to add a label with a title over 25 characters so that I have adequate information about the issue and I can choose another title.

Non-Functional Requirements

  • As a user, I would like the application to run without the internet so that I can rehearse (and run the application) in places that do not have internet access.
  • As a system/application, I would like to run entirely using local files so that an internet connection is not needed for the application to be run.
