Risks and Challenges

Risk Management Plan

During the planning portion of the project, I came up with a risk management plan. I listed out each potential risk, what steps were needed to mitigate each, the consequences if the risk became an issue, and a backup plan if the risks worsened. Then, I made sure to complete the tasks that I outlined for each risk to make sure that it did not become an issue. Thanks to this plan and work, none of the risks became issues.

Risks

Research MIDI Files

While I had a slight understanding of MIDI files before working on the project, I had never built anything that used MIDI files. I had never seen what a MIDI file looked like, the structure of the file, or what data was stored by it. I did some research about the file format before writing any code, so I understood how to work with the files before starting.

JavaScript Refresher

At the beginning of this project, it had been a year and a half since I had written any JavaScript code, since that was when I took my last JavaScript class. To make sure that I remembered the syntax and features of the language, I looked over some JavaScript tutorials to refresh my memory and prepare for making an application.

Determine and Learn MIDI JavaScript Library

I had never worked with MIDI files in code before, so I had to determine a library that could play and work with them. I found a few that seemed promising: Tone.js and WEBMIDI.js. Both could read MIDI events, but there was a key difference. Tone.js could generate tones that corresponded to MIDI notes. This means that MIDI files could be played directly in the browser, and that a MIDI keyboard could be played using only the browser. Also, I had Tone.js recommended to me by a graduated student who used it in the past. Before making the choice, I made proof of concepts with both libraries and looked at code examples. Tone.js seemed to have more features needed for the project that WEBMIDI.js did not have, so the choice was not very difficult.

Determine and Learn JavaScript Frontend Framework

I had never used a JavaScript framework for frontend development. While talking about the project with my professor, he recommended that I use Vue.js, as he had heard great things about it. I made some proof of concepts using Vue.js and read some tutorials, and decided to use it for the project. It ended up being a great solution, as I really enjoyed the framework's organization and simplicity.

Challenges

Most of the challenge came from all of the learning that was required to complete the project. I had to learn libraries, frameworks, tools, and more. As the project went on, I found better ways to, for example, create Vue components or utilize Tone.js to load and play MIDI files, thanks to research about various issues I was having during the project. I was able to complete the project's development, despite the various challenges.
